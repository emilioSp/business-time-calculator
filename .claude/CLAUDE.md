## Architecture

This is a focused TypeScript library that exports a single `BusinessTime` class for calculating working time between dates, excluding weekends and holidays.

**Source:** `src/BusinessTime.ts` — all business logic lives here (~323 lines). There are no sub-modules.

**Key types:**
- `DayOfWeek` — `'monday' | 'tuesday' | ... | 'sunday'`
- `Holiday` — string in `DD/MM` format (e.g., `'25/12'`)

**Constructor config:**
```typescript
new BusinessTime({
  businessTimezone: string,   // IANA timezone string
  businessDays: DayOfWeek[],  // Days that count as working days
  businessHours: number[],    // [startHour, endHour]
  holidays: Holiday[]         // Dates excluded from business time
})
```

**Public API surface:**
- `computeBusinessDaysInInterval / Hours / Minutes / Seconds / TimeInInterval` — interval calculations
- `addBusinessHoursToDate / addBusinessSecondsToDate` — forward date arithmetic
- `removeBusinessHoursFromDate / removeBusinessSecondsFromDate` — backward date arithmetic
- `isBusinessDay(datetime)` — predicate check
- `computeWorkingHours()` — hours per business day
- `_moveDateInBusinessTime({ datetime, moveBehind })` — aligns a datetime to the nearest business hour boundary (public for testing)

All date/time operations use **Luxon** (`luxon` v3) internally. The library is timezone-aware; pass datetimes as ISO strings and the class handles timezone conversion.

**Build output:** `dist/BusinessTime.js` + `dist/BusinessTime.d.ts` (excluded from git, generated by `npm run build`).

**Tests:** `src/BusinessTime.test.ts` uses `node:test` + `node:assert/strict`. Tests are grouped by method and use parameterized helpers (`testEachComputeTime`, `testEachMoveDateInBusinessTime`, `testEachIsBusinessDay`). The timezone is fixed to `Europe/Rome` to keep test expectations deterministic.
